Q1: Why might we consider using Git?

Answer: It helps us with version control, so we can revert to previous versions of our codebase if needed. It also 
acts as another way to store our codebase, so we can always delete or reset our local repository if needed. It also 
allows us to effective share code and have multiple people work on the same remote repository.

Q3: What is the difference between Merging and Rebasing?

Answer: Merging combines two branches by creating a merge commit that preserves the history of both branches, 
resulting in a non-linear commit graph showing parallel development. Rebasing rewrites a branch by replaying 
its commits on top of another branchâ€™s latest commit, producing a linear history. Unlike merging, rebasing 
does not preserve the original branch structure.
